Solución al Error 404 en GitHub Pages

Estás recibiendo un error 404 porque GitHub Pages no está configurado correctamente

Problema: Error 404 en https://MarcoData2.github.io

Este error ocurre porque:

  • El repositorio MarcoData2.github.io no existe, o
  • GitHub Pages no está habilitado en el repositorio, o
  • No hay un archivo index.html en la raíz del repositorio

Solución Paso a Paso

Sigue estos pasos para resolver el problema:

Verifica si el repositorio existe

Abre https://github.com/MarcoData2 y revisa si tienes un repositorio llamado MarcoData2.github.io.

Si no existe, necesitas crearlo.

Crea el repositorio (si no existe)

1. Haz clic en "+" → "New repository"

2. Nombra el repositorio: MarcoData2.github.io (EXACTO, con tu nombre de usuario)

3. Selecciona "Public"

4. Marca "Add a README file"

5. Haz clic en "Create repository"

Sube tu archivo HTML

1. En tu repositorio, haz clic en "Add file" → "Upload files"

2. Arrastra tu archivo HTML (debe llamarse index.html)

3. También puedes arrastrar carpetas con CSS, JS e imágenes si las necesitas

4. Haz clic en "Commit changes"

Habilita GitHub Pages

1. Ve a la pestaña "Settings" de tu repositorio

2. En el menú lateral, ve a "Pages"

3. En "Source", selecciona la rama "main"

4. Haz clic en "Save"

Espera y verifica

1. Espera unos minutos (puede tomar hasta 10 minutos para que se active)

2. Recarga https://MarcoData2.github.io

3. ¡Debería funcionar!

Verificación de Configuración Correcta

  • El repositorio se llama EXACTAMENTE: MarcoData2.github.io
  • GitHub Pages está habilitado en Settings → Pages
  • La rama seleccionada es "main" (o "master")
  • Hay un archivo index.html en la raíz del repositorio
  • Has esperado al menos 5 minutos después de la configuración

Si aún tienes problemas

Si después de seguir estos pasos aún ves el error 404:

  1. Revisa que el nombre del repositorio sea EXACTAMENTE igual a tu nombre de usuario + .github.io
  2. Asegúrate de que el archivo principal se llame index.html
  3. Prueba con un archivo HTML simple primero para verificar que funciona
  4. Espera más tiempo (a veces GitHub tarda hasta 20 minutos)
